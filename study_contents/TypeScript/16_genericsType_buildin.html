Genericsのビルドイン

TypeScriptにはジェネリック型を使用したビルトイン型が最初から組み込まれている。これらはTypeScript特有の機能であり、JavaScriptには存在しない。 このビルドイン型は特定な状況下で有用な機能なので、このような機能があるということを知っておいたほうがいい。 以下では2つ紹介するが、それ以外のビルドイン型はこちら

buildin型①Partial<T>

Partial型はジェネリックで指定した型のプロパティを、全て任意のプロパティに変更する型。 最終的にはジェネリック型で指定した型になるが、最初は任意のオプショナルプロパティにしておくことで、オブジェクトを生成した時点でプロパティに値が設定されていなくてもエラーにならない。 例えば以下のように、インターフェースを実装する関数を作成する。 関数の処理で、変数にインターフェース型を指定し、空の関数を値として格納すると、インターフェースに備わっているはずのプロパティの値がないのでエラーが出力される。

このような場合は、Partial型を使い、インターフェースを作成した時点ではプロパティをオプショナルにする。 注意 Partial<CourseGoal>はPartial型であり、GourseGoal型ではない。 そのため以下のコードのように、型をPartial型に変更したら、returnする値もCourseGoalにキャストしなくてはエラーになる。

Partial型の使い方 

// コースの目標を格納するインターフェース
interface CourseGoal {
  title: string;
  description: string;
  completeUntil: Date;
}

// CourseGoalインターフェースを作成する関数
function createCourseGoal(
  title: string,
  description: string,
  date: Date
): CourseGoal {
  // オブジェクトの戻り値にPartial型を設定
  let objCourseGoal: Partial<CourseGoal> = {};
  // それぞれの値をチェックしてからプロパティに値を入れたい
  // titleが10文字以下なら値を設定
  if (title.length <= 10) {
    objCourseGoal.title = title;
  } else {
    throw new Error('titleが10文字を超えています。');
  }
  // descriptionが300文字以下なら値を設定
  if (description.length <= 300) {
    objCourseGoal.description = description;
  } else {
    throw new Error('descriptionが300を超えています。');
  }
  // dateが現在より後なら値を設定
  const today = new Date();
  if (date.getTime() > today.getTime()) {
    objCourseGoal.completeUntil = date;
  } else {
    throw new Error('dateが過去日付になっています。');
  }
  // 値を戻す際キャストする(すべてのプロパティが追加されていることを保証する)
  return objCourseGoal as CourseGoal;
}

buildin型②Realonly<T>

Realonly型は値の追加、変更ができないように固定するための機能。変数だけではくオブジェクトにも使うことができる。 例えば配列などの定数を作成したい場合、参照型なのでconstにしてもpushやpopで値の追加や削除ができてしまう。 値を固定し、変更しようとした際エラーを出力したい場合はReadonly型を使用する。

Readonly型の使い方 

// 以下のコードではエラーにならない
const names = ['Max', 'Anna'];
names.push('Manu');
names.pop();

// Readonly型ならエラー検知できる
const names: Readonly<string[]> = ['Max', 'Anna'];
