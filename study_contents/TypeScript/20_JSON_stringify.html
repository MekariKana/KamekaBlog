JSON.stringifyについて

アクセサーデコレーターで、デコレーターの呼び出し前後で変更された値を比較しようとJSON.stringify()を使用したがうまく処理の前後が確認できなかった。 結局、差異を比較するための関数を作成したが、JSON.stringifyが変更前後を記録できなかった理由について調べてみた。

JSON.stringify()とは

JSON.stringifyはJavaScriptの"値"をJSON形式の文字列に変換する関数。 つまりJSON.stringifyの引数の値を「その瞬間に文字列に変換する」だけ。だから冒頭で説明したように、変更前後の差異を確認するのに変更前の値をJSON.stringifyで変数に格納すればいいと思った。 しかし、この関数の欠点は「プログラムとしての動作」つまり関数・プロトタイプ(classの情報)・シンボルなどは保存できない仕様である。 注意点 関数は文字列化できない ->関数は「データ」ではなく「処理(実行するためのコード)」なので、JSON(データ交換用フォーマット)では表現できない

関数が表示されない例 

console.log(
    JSON.stringify({
        name: "Taro",
        greet: function() { console.log("hi"); }
    })
);

出力結果は以下のとおり、nameのみ出力し、functionの値を持っているgreetプロパティは表示されない。

JSONが扱えるデータの種類

JSONで扱えるデータは以下の6種類のみ ・number ・string ・boolean ・null ・array ・object(ただし値はプリミティブ型・配列・オブジェクトのみ) 上記以外の関数やundefined、symbolはJSONでは表現できない。 Point JSON.stringifyが文字列変換できるのは"静的な値"のみ
