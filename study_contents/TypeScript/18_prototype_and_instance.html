コンストラクタ関数(クラス本体)とプロトタイプについて

デコレーターを勉強しているとき、プロパティデコレーターでこの2つの用語がでてきたので、別途まとめる。

プロトタイプとは

クラスに定義された共通処理(メソッドなど)にどのようなものがあるか記したメソッド一覧のようなもの。また、インスタンス化して初めて共通処理が使えるようになるので、メソッドはインスタンスした実体のほうに属する。 プロパティはインスタンス化したときコンストラクターで値が設定される。インスタンス各々が違う値を持つので、プロトタイプの情報には含まれない。

コンストラクタ関数(クラス本体)とは

コンストラクタ関数とは、newしてオブジェクトを作る「関数」そのもの(クラス本体)のこと。インスタンス化して初めて動作するので、関数(クラス)自体は実態(オブジェクト)ではない。 しかし、クラスの情報はインスタンス化前でもJavaScriptがコードを読んだ時点で解釈してる(メモリ上にクラスの構造が作られている)。 以下の例の場合、staticプロパティはインスタンス化した実態ではなくクラス本体に属しており、評価されたクラス構造に含まれている。だから、staticプロパティにアクセスするときも、[クラス名.staticプロパティ]の構文で呼ぶ。

プロトタイプとインスタンス関数の違い 

function Log(target: any, propertyName: string | symbol) {
    console.log("Logデコレーター - start");
    console.log(target, propertyName);
}

class Student {
    @Log
    name: string = ""; // インスタンス(new)したら実体化(インスタンスに属する)
    @Log
    static schoolName = "青空小学校"; // クラスで共有(クラスに属する)

    constructor(name: string) {
        this.name = name;
    }

    greet() {
        console.log("おはようございます。" + this.name + "です。");
    }
}

// インスタンスプロパティとメソッドを出力
const s1 = new Student("はなこ");
console.log(s1.name);
s1.greet();

// staticプロパティ出力
console.log(Student.schoolName);

出力結果は以下のとおり。 一番上のインスタンスプロパティのtargetはprototypeが出力され、次のstaticプロパティのtargetはStudentクラス本体が出力されている。
