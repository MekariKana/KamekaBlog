VSでのDebug方法

環境設定

1.「JavaScript Debugger」の拡張機能を追加する。

2.[tsconfig.json]ファイルの"sourceMap"をtrue(有効)に設定する。 -> 実際にデバック実行するのはコンパイルされたjsファイルなので、どのtsのコードがどのjsコードと対応しているコードを知るため、mapファイルが必要。

デバックを実行する

1.一番左のアイコンの虫マークをクリックし、デバック画面を開く。

2.「実行とデバック」をクリック

3.デバック実行するブラウザを選択(ここではChrome)

4.[launch.json]ファイルが自動で作成される。以下のように設定したら、[tsc]コマンドでコンパイルする。 -> このファイルは、デバック実行の時どのようにChromeを起動するか設定するためのファイル。 ・url：ポートは初期設定で8080になっている。ここでは開発用サーバ(lite-server)が3000で起動しているためポートを3000に修正する。 ・webRoot：{workspaceFolder}とはプレジェクトのルートフォルダを指すプレスフォルダ(環境変数)。このプロジェクトのルートフォルダがサーバのルートであることを前提にするという設定になっているので、変更しなくてOK。

※サーバーのルートとは、Webサーバが公開する基点となるフォルダ。ルートを変えたら、サーバ起動時に読み込むフォルダが変わる。 Webサーバはブラウザに見せてもいい基準点となるルートフォルダ決め、そこからファイルを探す。 プロジェクトのルートは、Next.jsやCreate React Appなどのツールやlite-serverなどサーバによって決められているので、都度調べて確認する。 例）サーバのルートを"my-project/public"に設定した場合 -> "http://localhost:3000/index.html" -> my-project/public/index.htmlが返る src/main.jsはルートフォルダ配下に含まれないので、直接アクセスできない。index.htmlからmain.jsを読んでいる場合は404エラーになる。

フォルダ階層 

my-project/
  ├─ public/
  │    ├─ index.html
  │    └─ css/
  │         └─ style.css
  └─ src/
       └─ main.js

※ 開発用サーバのurl(ポート)の確認方法 1. [package.json]ファイルの"scripts"を確認。"start"や"run dev"など開発用サーバーを呼ぶコマンドに、どのサーバが設定されているか確認。

2.[npm start]コマンドで開発用サーバを起動。以下のようにターミナルに接続したURLが表示されるのでポートを確認する。

terminal 

[BS] Access URLs:
 ----------------------------------
       Local: http://localhost:3000
    External: http://192.168.xx.xx:3000
 ----------------------------------

-> lite-serverはデフォルトではポート3000に接続される。別途ポートを設定する場合は、bs-config.json または bs-config.jsというファイルで設定する。 この設定ファイルが存在しない場合は、デフォルト設定で動く。 設定ファイルでポートを指定する場合は、以下のようにbs-config.jsonを作成する。

bs-config.json 

{
  "port": 4001,
  "server": { "baseDir": "./src" }
}

デバックツールの画面説明

変数：今のスコープでアクセスできる変数の中身を確認できるところ ウォッチ式：任意の式を挿入し、実行結果を監視することができるところ コールスタック：関数の呼び出し階層を確認するところ ブレークポイント：コードの実行が一時停止する場所を確認するところ

デバックツールの使い方

1.ブレークポイントを設定する(ソースの行数が書いてある左のスペースをダブルクリックすると、赤い丸印が表示される。) 2.[npm start]コマンドでlite-serverを立ち上げる 3.変数スペースの上の緑の▶︎をクリックしてデバックをスタート 4.ブレークポイントで一時停止するので、右上のコントロールパネルを操作して処理を進める。 ＜コントロールパネルの機能＞ |▹：処理を再開する ↷（ステップオーバー）：現在の関数の実行を1行進める ↓（ステップイン）：関数の呼び出し階層の中に入って、更に1階層深い関数をデバックする ↑（ステップアウト）：1つ上の呼び出し階層に戻るまで実行を進める ↺：再起動 ☐：切断
